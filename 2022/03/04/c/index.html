函数类型

虚函数

定义一个函数为虚函数，不代表函数为不被实现的函数。

定义他为虚函数是为了允许用基类的指针来调用子类的这个函数。里氏代换原则就是任何用子类的地方都可以用父类来代替，这样在编码的时候只需要使用父类类型，达到多态的效果。

class A  
{  
public:  
    virtual void foo()  
    {  
        cout<<"A::foo() is called"<<endl;  
    }  
};  
class B:public A  
{  
public:  
    void foo()  
    {  
        cout<<"B::foo() is called"<<endl;  
    }  
};  
int main(void)  
{  
    A *a = new B();  
    a->foo();   // 在这里，a虽然是指向A的指针，但是被调用的函数(foo)却是B的!  
    return 0;  
}

纯虚函数

纯虚函数是在基类中声明的虚函数，它和虚函数的区别在于：1.它在基类中没有定义 2.要求任何派生类都要定义自己的实现方法。在基类中实现纯虚函数的方法是在函数原型后加“=0”,如：$virtual void funtion1()=0$。
这就类似JAVA里面的接口，实现/继承的子类必须实现其方法

#include <iostream>
using namespace std;
class Interface{
public:
    virtual void show()=0;
};
class Implements1:public Interface{
public:
    void show(){
        cout<<"There is Implements1"<<endl;
    }
};
class Implements2:public Interface{
public:
    void show(){
        cout<<"There is Implements2"<<endl;
    }
};
int main(){
    Interface *interface=new Implements1();
    interface->show();
    return 0;

友元

friend 修饰函数名或者类名，意思就是这个函数或者类是我的朋友，它们可以访问我的私有成员

//这个案例就是vip可以享受商城的低价，因为vipPrice是私有的，正常类得不到这个价格，但是商城把vip设为友元，那么vip就可以知道这个价格了
class Vip {
    public:
        goShop(ShoppingMall * sp);
}

class ShoppingMall {
    public:
        int Price;
    private:
        int vipPrice;
        friend class Vip;
}

内联函数

调用时并不通过函数调用的机制而是通过将函数体直接插入调用处来实现的，这样可以大大减少由函数调用带来的开销，从而提高程序的运行效率,所以inline函数一般用于比较小的，频繁调用的函数，这样可以减少函数调用带来的开销，如果函数较大那么会增大编译后文件。只需要在函数返回类型前加上关键字inline，即可将函数指定为inline函数。

宏

宏是可以带参数的，它在形式上和函数非常相似，但宏仅仅是字符串替换，不是按值传递。
如：$#define mutiple(n) nn$，当n=4时，$mutiple(n)=44=16$，当$n=4+1，mutiple(n)=4+1*4+1=9$

与内联函数的区别

  • 内联函数在编译时展开，而宏在预编译时展开

  • 在编译的时候，内联函数直接被嵌入到目标代码中去，而宏只是一个简单的文本替换。

  • 内联函数可以进行诸如类型安全检查、语句是否正确等编译功能，宏不具有这样的功能。

  • 宏在定义时要小心处理宏参数，一般用括号括起来，否则容易出现二义性。而内联函数不会出现二义性。

转换运算符

一个指向字符串的指针是如何地与一个指向整数的指针或一个指向其他自定义类型对象的指针有所不同呢？从内存需求的观点来说，没有什么不同！它们三个都需要足够的内存（并且是相同大小的内存）来放置一个机器地址。指向不同类型之各指针间的差异，既不在其指针表示法不同，也不在其内容（代表一个地址）不同，而是在其所寻址出来的对象类型不同。也就是说，指针类型会教导编译器如何解释某个特定地址中的内存内容及其大小。

  • 向上转型（隐式转型）：将派生类转为基类，安全并且自动完成
  • 向下转型：基类转派生，如基类本身就指向的派生类那么转换就是安全的，反之不安全

typedef

  • 定义一种类型的别名，而不只是简单的宏替换。可以用作同时声明指针型的多个对象。比如：
    typedef char* PCHAR; 
PCHAR pa, pb => char *pa, *pb
  • 当要跨平台时，浮点精度不一样，那么只需要定义一个新类型，不同平台改typedef
    如

一些区别

struct与class区别

class 和 struct 的用法是完全相同的，但只有一个本质区别，如下：

  • 在使用 struct 定义类时，所有成员的默认访问级别为 public；
  • 在用 class 定义类时，所有成员的默认访问级别为 private；

const和constexpr

  • const不能被赋值，包括构造函数。唯一的方式是初始化列表,初始化列表先于构造函数的函数体执行,列表一定是在对象创建的同时执行，先于函数体。
class constTest{
    private:
        int var1;
        int var2;
        const int var3;
         constTest() : var1(v1),var2(2),var3(3)
        {
            ...
            ...
        }
}
  • constexpr是修饰一个常量表达式
template <int N>
class list
{ };

constexpr int sqr1(int arg)
{ return arg*arg; }

int sqr2(int arg)
{ return arg*arg; }

int main()

static和const

static表示的是静态的。类的静态成员函数、静态成员变量是和类相关的，而不是和类的具体对象相关的。即使没有具体对象，也能调用类的静态成员函数和成员变量。一般类的静态函数几乎就是一个全局函数，只不过它的作用域限于包含它的文件中。

const数据成员 只在某个对象生存期内是常量，而对于整个类而言却是可变的。因为类可以创建多个对象，不同的对象其const数据成员的值可以不同。所以不能在类的声明中初始化const数据成员，因为类的对象没被创建时，编译器不知道const数据成员的值是什么。

通俗的来说：
static就表示与具体哪个对象无关，但是每个对象都有
const就是表示出生之后就没法改变，但是每个对象可以不同

enable_shared_from_this

在一个类B被share_ptr管理的前提下，B中有个get方法可以将自己传递出去，这个时候不能自己构造一个share_ptr指针如：
std::shared_ptr(this)，这回导致两个share_ptr指向同一个对象，但指针都认为自己独享这个对象，所以在这个时候就需要让类A继承 std::enable_shared_from_this 然后调用 shared_from_this()

nodiscard

C++17引入的标记符，语法为：[[nodiscard]]，如：[[nodiscard]] int func(){return 1;};
被标记的方法需要有接受值，反之编译器会警告，如：正常应该是int x = func()，而不是func()
